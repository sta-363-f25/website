Predicting handwritten numbers

Author

Lucy D’Agostino McGowan 

library(tidyverse)
library(tidymodels)
load("data/zip.train.Rdata")
load("data/zip.test.Rdata")

This dataset consists of normalized handwritten digits, automatically scanned from envelopes by the U.S. Postal Service. The ultimate objective is to be able to predict which number is written so that the zip code can be automatically determined. The original scanned digits are binary and of different sizes and orientations; the images here have been deslanted and size normalized, resulting in 16 x 16 grayscale images (Le Cun et al., 1990). This data is obtained from the ElemStatLearn package.

Let’s see what kind of data this is.

class(zip.train)
[1] "matrix" "array"

It looks like this is a matrix. Let’s look at the the first 5 rows and 5 columns.

zip.train[1:5, 1:5]
     [,1] [,2] [,3] [,4]   [,5]
[1,]    6   -1   -1   -1 -1.000
[2,]    5   -1   -1   -1 -0.813
[3,]    4   -1   -1   -1 -1.000
[4,]    7   -1   -1   -1 -1.000
[5,]    3   -1   -1   -1 -1.000

It looks like the first column contains the digit and the subsequent columns indicate values for the 256 pixels. Let’s tidy this up a bit.

zip_train <- zip.train %>%
  as_tibble(.name_repair = ~ c("label", glue::glue("pix_{1:256}")))

zip_test <- zip.test %>%
    as_tibble(.name_repair = ~ c("label", glue::glue("pix_{1:256}")))

Let’s plot a few. This code is adapted from a blog post by David Robinson.

pixels_gathered <- zip_train %>%
  slice(1:25) %>%
  mutate(instance = row_number()) %>%
  gather(pixel, value, -label, -instance) %>%
  tidyr::extract(pixel, "pixel", "(\\d+)", convert = TRUE) %>%
  mutate(pixel = pixel - 1,
         x = pixel %% 16,
         y = 16 - pixel %/% 16)

ggplot(pixels_gathered, aes(x, y, fill = value)) +
  geom_tile() +
  facet_wrap(~ instance + label)

Predict 0

We are interested in creating a prediction model to determine whether the handwritten number is a 0.

First we create a recipe

recip <- zip_train %>% 
    recipe() %>% 
    update_role(starts_with("pix"), new_role = "predictor") %>%
    update_role(label, new_role = "outcome") %>%
    step_mutate(label = as.factor(label == 0)) ## Set the outcome to when the written number is equal to 0

Then we prep our recipe.

prepped <- recip %>% 
    prep(retain = TRUE)

Then we create our training data and testing data

train <- prepped %>% 
    juice()

test <- prepped %>% 
    bake(new_data = zip_test)

Now we specify our model

We are fitting a decision tree.

tree_fit <- fit(
    decision_tree(mode = "classification", tree_depth = 3),
    formula = formula(prepped),
    data = train,
    engine = "rpart")

Assess the model

results <- tibble(
    truth = test$label,
    estimate = predict(tree_fit, test) %>% 
      pull(.pred_class)
  )

metrics(results, truth = truth, estimate = estimate) %>% 
  knitr::kable()
.metric .estimator .estimate
accuracy binary 0.9362232
kap binary 0.7861534

Create a confusion matrix

What is the false positive rate? What is the false negative rate?

conf_mat(results, truth = truth, estimate = estimate)[[1]] %>% 
  as_tibble() %>% 
  ggplot(aes(Prediction, Truth, alpha = n)) + 
  geom_text(aes(label = n), alpha = 1, size = 10) +
  labs(title = "Confusion matrix")
